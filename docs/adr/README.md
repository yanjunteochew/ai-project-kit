# Architecture Decision Records (ADRs)

This directory contains Architecture Decision Records documenting significant technical decisions.

## Purpose

ADRs capture:
- Context and problem statement
- Proposed solution
- Alternatives considered
- Consequences and trade-offs
- Decision status and date

## Format

Each ADR should be a separate markdown file named: `NNNN-brief-title.md`

Example: `0001-use-nodejs-for-runtime.md`

## Status

ADRs can have the following status:
- **Proposed** - Under consideration
- **Accepted** - Decision made
- **Superseded** - Replaced by a newer ADR
- **Deprecated** - No longer applicable
